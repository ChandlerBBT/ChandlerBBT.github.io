第 15 章 分层模型的魅力

欢迎来到第四单元！

第四单元的主题是分层。日常中“我的工作场所等级森严”这样的表达可能带有贬义，但“我的贝叶斯模型是分层模型”往往是一件好事！分层模型通过容纳分层（或称为分组）数据，极大地扩展了建模工具箱的灵活性。例如，我们的数据可能包括：

  • 抽样的多所学校，以及每所学校内多名学生的数据 \(y\)；或者
  • 抽样的多个实验室，以及每个实验室内多个实验的数据 \(y\)；或者
  • 抽样的一组人群，我们对其随时间重复观测信息 \(y\)。

忽略这种底层分组结构会违反第三单元模型背后的独立数据假设，进而可能产生误导性结论。在第四单元，我们将探索如何在模型中融入这种分层结构：

  • 第15章：为什么你应该对分层建模感到兴奋
  • 第16章：不含预测变量的（正态）分层模型
  • 第17章：含预测变量的（正态）分层模型
  • 第18章：非正态分层模型
  • 第19章：增加更多层级

在开始之前，我们想说明，这里用“分层”作为概括分组结构模型的统称。在文献中，这类模型被称为多水平模型、混合效应模型或随机效应模型。这些术语可能令人困惑，特别是有些人将它们互换使用，而另一些人用它们对分组结构模型进行细微区分，有点像“汽水”“苏打水”和“可乐”的叫法之分。我们在此避免这种困惑，但指出这一点，以便你在其他场合能建立起联系。

在转向分层模型之前，理解我们为何要这样做至关重要。在第 15 章，我们的唯一目的就是回答这个问题。我们不会构建、解释或模拟任何分层模型，而只是为这些技术的重要性搭建一个论证，以填补现有贝叶斯方法的一些缺口。为此，我们将探索 cherry_blossom_sample 数据集（来自本书配套数据资源，此处简称为 running）：

  • 考察分层贝叶斯模型所提供的部分合并的优势。

15.1 完全池化

我们将采用完全池化方法开始分析跑步用时与年龄之间的关系：将 36 名跑者的全部 252 个观测值合并为一个信息池。 在此过程中，请注意这种关系似乎较弱——每个年龄段的跑步用时存在相当大的变异，并且随着年龄增长没有明显的趋势：

# 绘制散点图：年龄与净时间的关系
sns.scatterplot(data=running, x="age", y="net")
plt.show()

图 15.2：每次比赛结果的净跑步用时与年龄的散点图。

为了更好地理解这种关系，我们可以构建一个熟悉的正态回归模型，将跑步用时表示为年龄的函数。 令 \(Y_i\) 和 \(X_i\) 分别表示数据集中第 \(i\) 个观测值的跑步用时和年龄，该模型的数据结构为：

\[\begin{equation} Y_i | \beta_0, \beta_1, \sigma \stackrel{ind}{\sim} N\left(\mu_i, \sigma^2\right) \;\; \text{ with } \;\; \mu_i = \beta_0 + \beta_1X_i . \tag{15.1} \end{equation}\]

我们可以像往常一样模拟这个完全池化模型，这里使用弱信息先验：

import pymc as pm
import arviz as az
import numpy as np
import pandas as pd

# 假设 running 数据集已加载为 pandas DataFrame
# running = pd.read_csv('running.csv')

with pm.Model() as complete_pooled_model:
    # 先验设置
    intercept = pm.Normal('intercept', mu=0, sigma=2.5)
    slope = pm.Normal('slope', mu=0, sigma=2.5)
    sigma = pm.Exponential('sigma', lam=1)
    # 线性预测
    mu = intercept + slope * running['age'].values
    # 观测模型
    net_obs = pm.Normal('net_obs', mu=mu, sigma=sigma, observed=running['net'].values)
    # 采样
    trace = pm.sample(draws=5000, tune=5000, chains=4, random_seed=84735)

模拟结果与我们先前的观察一致，即跑步用时与年龄之间存在较弱的关系。 后验中位数模型表明，年龄每增加一岁，跑步用时往往仅增加 0.27 分钟。 并且，\(\beta_1\) 的 80% 后验可信区间跨过了 0（-0.3, 0.84），表明这种关系并不显著：

# 后验汇总统计
import arviz as az

# 假设 trace 已从 complete_pooled_model 获得
summary_df = az.summary(trace, var_names=['intercept', 'slope'], hdi_prob=0.8, round_to=3)
print(summary_df[['mean', 'sd', 'hdi_5%', 'hdi_95%']])

好吧。我们本不想第一个说出来，但这似乎有点奇怪。 我们自身的经验并不支持我们会一直保持同样速度的观点。 事实上，请看图 15.3。 如果我们分别检查 36 名跑者每个人的跑步用时与年龄的关系（灰色线条），几乎所有人都随着时间的推移变慢了，并且变慢的速度比后验中位数模型（蓝色线条）所显示的要快。

在图 15.4 中，我们聚焦于三名跑步者的细节。尽管完全池化模型（蓝色线）在某些情况下表现尚可（例如跑步者 22），但它远非普遍适用。不同跑步者的总体速度以及速度随年龄的变化差异相当大。例如，跑步者 20 往往跑得比平均慢而且随着年龄增长，速度下降得更快。跑步者 1 则一直跑得比平均快。

图 15.4：3 名受试者的跑步时间与年龄的散点图，以及后验中位数模型（蓝色）。

就这些观察而言，我们的完全池化模型确实不准确。如图 15.5 所示，完全池化模型将所有跑步时间的观测值（\(y_1, y_2, \ldots, y_n\)）归入一个总体或一个“池”中。这样做，它作出了两个假设：

  1. 每个观测值都与其他观测值相互独立（这是第三单元模型背后的普遍假设）；以及
  2. 关于个体跑步者的信息与我们的跑步时间与年龄模型无关。

我们现在观察到这些假设是不合适的：

  1. 虽然一个跑步者的观测值可能独立于另一个，但同一个跑步者内部的观测值是相关的。也就是说，跑步者在前一场比赛中的速度能告诉我们他们在下一场比赛中可能会跑多快。
  2. 就跑步时间与年龄的关系而言，人天生就各不相同。

这些假设违背带来了显著后果：我们的完全池化模型未能捕捉到人们往往随年龄增长而变慢这一事实。 虽然我们是在樱花赛跑的特定背景下探讨了完全池化的缺点，但这些缺点具有普遍性。

图15.5：完全池化模型的一般框架。

完全池化方法的缺点

使用完全池化方法分析具有组结构的数据有两个主要缺点：

  1. 我们违反了独立性假设；进而，
  2. 我们可能会对关系本身及其显著性得出误导性结论。

15.2 不池化

完全池化模型失败了，我们转向另一个极端。 不池化（no pooling）方法不把所有人混在一起、忽视任何关于跑者的信息，而是分别考虑我们的 \(m = 36\) 位跑者。 这个框架由图 15.6 表示，其中 \(y_{ij}\) 表示跑者 \(j\) 的第 \(i\) 次观测。

图15.6：不池化模型的一般框架。

这个框架也意味着不池化方法为每位跑者构建一个单独的模型。 具体来说，设 \((Y_{ij}, X_{ij})\) 表示跑者 \(j\) 在其第 \(i\) 次比赛中的观测跑步时间和年龄。 那么跑者 \(j\) 的跑步时间与年龄的正态线性回归模型的数据结构为：

\[\begin{equation} Y_{ij} | \beta_{0j}, \beta_{1j}, \sigma \sim N\left(\mu_{ij}, \sigma^2\right) \;\; \text{ 其中 } \;\; \mu_{ij} = \beta_{0j} + \beta_{1j} X_{ij}. \tag{15.2} \end{equation}\]

该模型允许每位跑步者 \(j\) 拥有独特的截距 \(\beta_{0j}\) 和年龄系数 \(\beta_{1j}\)。或者，在跑步的语境下，无池化模型反映了有些人天生跑得比其他人快（因此 \(\beta_{0j}\) 不同）以及速度随时间的变化因人而异（因此 \(\beta_{1j}\) 不同）。虽然我们不会实际实现这 36 个无池化模型，但如果采用模糊先验，得到的个体后验中位数模型将与下图类似：

import seaborn as sns
import matplotlib.pyplot as plt

g = sns.FacetGrid(examples, col="runner")
g.map(sns.regplot, "age", "net", ci=None, truncate=False, scatter=True)
g.set(xlim=(52, 62))
plt.show()

图 15.7：3 名被试的跑步时间与年龄散点图，以及被试特异性趋势（蓝色）。

这初看似乎不错。跑步者特异性模型捕捉到了跑步者特异性的趋势。然而，无池化方法有两个显著的缺点。首先，假设你计划在未来几年内每年参加樱花长跑比赛。根据我们三个示例案例的无池化结果，你预计你的跑步时间会是多少？你被难住了吗？如果没有，你应该犯难。无池化方法无法帮助你回答这个问题。因为这些模型是针对我们样本中的 36 个个体定制的，得到的 36 个模型无法可靠地外推到这些个体之外。再考虑第二个小问题，做个小测验。67

重新查看图 15.7 中的跑步者 1。如果他们在 62 岁时第六次参赛，距离他们最近的数据点已有 5 年，你预计他们的 net 跑步时间会是多少？

  1. 低于 75 分钟
  2. 75 到 85 分钟之间
  3. 高于 85 分钟

如果你使用非合并模型来回答这个问题，你的答案会是 a。跑步者 1 的模型表明，他们随着年龄增长越来越快，到 62 岁时跑步时间应该低于 75 分钟。然而，这一非合并结论存在于真空中，只考虑了跑步者 1 的数据。从其他 35 名跑步者中，我们观察到大多数人会随着年龄增长变慢。完全忽略这一信息是不合理的，尤其是我们只有区区五场比赛的跑步者 1 样本量（因此没有理由忽视额外数据！）。一个更合理的预测可能是选项 b：尽管跑步者 1 可能不会保持如此陡峭的下降趋势，但他们很可能仍是一名快速跑步者，比赛时间在 75 到 85 分钟之间。同样，这才是合理的结论，而不是我们仅使用非合并模型得出的结论。虽然我们在樱花赛跑的具体背景下探讨了非合并方法的缺点，但这些缺点具有普遍性。

非合并方法的缺点

采用非合并方法分析分组结构数据有两个主要缺点：

  1. 我们无法可靠地将特定于组的非合并模型推广或应用于样本之外的组。
  2. 非合并模型假设一个组不包含关于另一个组的相关信息，因此忽略了可能有价值的信息。当每组观测数量较少时，这一点尤其重要。

15.3 分层数据

樱花赛跑数据给我们带来了一个新的挑战：分层数据。这种层次结构的一般结构如图 15.8 所示。该层次结构的顶层是所有跑步者的总体，而不仅仅是我们样本中的跑步者。下一层是我们的 36 名跑步者，是从这个总体中抽取的。在本章之前的每个分析中，到这里就停止了。然而在 running 数据中，层次结构的最后一层包含每名跑步者的多个观测值，通常称为重复测量或纵向数据。

分层（或分组）数据结构在实践中很常见。这些并不总是像 running 数据那样源于对个人的多次观测。在分析新的教学技术时，我们的组可能是学校，而我们的观测是每所学校内的多名学生：

  school_id tech score
1         1    A    90
2         1    A    94
3         1    B    82
4         2    A    85
5         2    B    89

在分析天然驱蚊剂时，我们的组可能是一组实验室样本，而我们的观测是每个实验室内的多个实验：

  lab_id repellant score
1      1         A    90
2      1         A    94
3      1         B    82
4      2         A    85
5      2         B    89

以这种分层方式收集数据有充分的理由。首先，在时间、金钱、后勤和其他资源方面，它通常更具可持续性。例如，尽管两种数据收集方法都会产生 100 个数据点，但分别在 5 所不同学校各采访 20 名学生所需的资源比在 100 所不同学校各采访 1 名学生要少。分层数据还能深入了解两个重要特征：

  1. 组内变异性
    每组内部多个观测值之间的变异程度本身就很有趣。例如，我们可以考察个人跑步成绩逐年的一致性如何。

  2. 组间变异性
    分层数据还使我们能够考察组与组之间的变异性。例如，我们可以考察跑步模式因人而异的程度。

15.4 分层模型中的部分合并（partial pooling）

我们现有的贝叶斯建模工具箱提供了两种分析分层数据的方法。 我们可以完全忽略分组结构，将所有组混在一起，通过完全合并（complete pooling）假设一个模型普遍适用（图 15.5）。 或者我们可以分别分析每个组，通过不合并（no pooling）假设一个组的模型不能告诉我们关于另一个组的任何信息（图 15.6）。 在“没有个性空间”和“人人为己”这两种态度之间，分层模型通过部分合并（partial pooling）提供了一个受欢迎的中庸之道（图 15.8）。 一般来说，分层模型秉承以下思想：尽管每个组都是独特的，但由于它们来自同一个总体，所有组都是相互关联的，因此可能包含关于彼此的有价值信息。我们在此深入探讨这一思想，并将在第 16 章实际构建分层模型。

图 15.8：分层模型的通用框架。

图 15.9 比较了针对三位示例跑者以及你（一位我们尚未观察到的新跑者）的跑步时间随年龄变化的分层模型、不合并模型和完全合并模型。 考虑我们的三位示例跑者。 尽管利用了相同的数据，我们的三种技术为这些跑者产生了不同的后验中位数模型。 与完全合并模型（黑色）不同，分层模型（虚线）允许每位个体跑者拥有独特的后验趋势。 这些趋势检测到了跑者 1 是我们的示例跑者中最快的事实。 但与不合并模型（蓝色）不同，针对特定跑者的分层模型不会忽略其他跑者的数据。 例如，考察跑者 1 的结果。 仅基于该跑者的数据，不合并模型假设他们的下降趋势将继续。 相反，在只有跑者 1 的五个数据点，却有来自其他 35 位跑者的有力证据表明人们通常会变慢的情况下，分层趋势的斜率被向上拉动了。 一般来说，识别到一位跑者的数据可能包含关于另一位的有价值信息，分层模型趋势会偏离不合并的个体模型，并趋向完全合并的通用模型。

图 15.9：不合并模型（蓝色）、完全合并模型（黑色）和分层模型（虚线）针对三位示例跑者以及你的跑步时间随年龄变化的后验中位数模型。

最后，考虑为你这位新跑者的建模结果。 注意你的面板中没有蓝线。 由于你不在原始的 running 样本中，我们不能使用不合并模型来预测你的跑步趋势。 相反，由于它们没有忽略我们的跑者所来自的更广泛总体，完全合并模型和分层模型都可以用于预测你的趋势。 但结果不同！ 由于完全合并模型未能检测到 50 多岁的跑者往往随时间变慢的事实，你的后验中位数模型（黑色）的斜率接近零。 虽然这是一个美好的想法，但分层模型可能提供了一个更现实的预测。 基于所有 36 位跑者的数据，你可能会变慢一点。 此外，你预期变慢的速度与我们 36 位跑者的平均速度相当。

15.5 章节总结

在第 15 章中，我们强调了将另一技术——分层模型——纳入贝叶斯工具箱的重要性。 我们着眼于大局而非细节，探讨了研究分组数据的三种方法：

  • 完全池化模型将所有数据点合并在一起，假设它们是独立的，并且一个通用模型适用于所有组。这可能会对所关注的关系及其重要性产生误导性结论。简而言之，完全池化模型的态度是“没有个体差异的空间”。

  • 非池化模型为每个组建立单独的模型，假设一个组的模型不能告诉我们任何关于另一个组的信息。这种方法未能充分利用数据，并且无法推广到样本外的组。简而言之，非池化模型的态度是“各组自顾自”。

  • 部分池化模型（即分层模型）提供了一种折衷方案。它们承认，尽管每个组都可能有自己的模型，但一个组可以提供关于另一组的有价值信息。也就是说，“让我们互相学习，同时保持个体独特性”。
