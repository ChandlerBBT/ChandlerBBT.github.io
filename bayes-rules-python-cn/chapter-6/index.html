第 6 章 近似后验分布

欢迎来到第2单元！

第2单元是一座关键的桥梁，将第1单元的基本概念应用到第3单元及以后更复杂的模型环境中。在第1单元，我们学会了像贝叶斯学派一样思考，并秉承这种精神构建了一些基本的贝叶斯模型。此外，通过贝叶斯规则将这些模型运转起来，我们能够在数学上精确指定相应的后验分布。那样的日子已经过去了。虽然目前只是假设，但总有一天（从第9章开始），我们感兴趣分析的模型会变得过于复杂，无法用数学精确指定。别担心——数据分析师可不是那种在未知面前就举手投降的人。当我们无法知道或精确指定某个东西时，我们就会去近似它。在第2单元，我们将探索马尔可夫链蒙特卡洛模拟技术，用来近似那些原本难以企及的后验模型。

无论我们能否精确指定还是必须近似后验模型，接下来我们都必须能够理解并应用结果。为此，我们在第1单元学习了如何使用中心趋势、变异性等模型特征来描述我们的后验认知。然而在实践中，我们通常希望进行更深入的后验分析。这个探究“这都意味着什么？！”的过程围绕三大要素展开，我们将在第2单元探讨：后验估计、假设检验和预测。

学习偶尔需要跳跃。你已经跳跃了几次。从第2章到第3章，你完成了从使用简单的离散先验到对比例\(\pi\)使用连续Beta先验的跳跃。从第3章到第5章，你完成了从构建Beta-二项模型到能应用于更广泛场景的贝叶斯模型族的跳跃。每一次跳跃，你的贝叶斯工具箱都变得更灵活、更强大，但代价是背后的数学变得稍微复杂了一些。随着你继续在更复杂的环境中推广你的贝叶斯方法，这种复杂性还将继续增加。

考虑米歇尔的总统竞选。在第3章，你基于该州的民调数据，构建了米歇尔在明尼苏达州支持率的模型。你可以继续完善对米歇尔当选总统可能性的分析。首先，你可以对米歇尔在每一个州和华盛顿特区的支持率进行建模。更好的是，这个模型还可以纳入过去各州投票趋势和人口统计数据。取舍之处在于，增加模型的灵活性也会让它变得更复杂。你原来只依赖一个参数\(\pi\)的明尼苏达模型，即米歇尔在那里的支持率，而新模型则依赖几十个参数。这里，令\(\theta = (\theta_1, \theta_2, \ldots, \theta_k)\)表示贝叶斯模型所依赖的\(k \ge 1\)个参数的通用集合。在你不断发展的米歇尔选举概率贝叶斯模型中，\(\theta\)包含了51个代表各州支持率的参数，以及定义米歇尔在选民中的支持率、州级人口统计数据和过去投票趋势之间关系的多个参数。因此，我们的后验分析依赖于在给定一组关于州级民调、人口统计和投票趋势的观测数据\(y\)的条件下，构建\(\theta\)的后验概率密度函数（pdf），

\[f(\theta | y) = \frac{f(\theta)L(\theta | y)}{f(y)} \propto f(\theta)L(\theta | y) .\]

虽然这个公式看起来很熟悉，但背后潜藏着复杂性。由于\(\theta\)是一个包含\(k\)个元素的向量，\(f(\theta)\)和\(L(\theta|y)\)是复杂的多元函数。而且，如果我们珍惜时间，就别想计算所有可能\(\theta\)下的归一化常数\(f(y)\)，它是一个棘手的多重积分（或求和），可能不存在解析解：

\[f(y) = \int_{\theta_1}\int_{\theta_2} \cdots \int_{\theta_k} f(\theta)L(\theta | y) d\theta_k \cdots d\theta_2 d\theta_1 .\]

幸运的是，当贝叶斯后验分布无法或极其困难精确指定时，我们并非无计可施。我们只需改变策略：与其精确指定后验分布，不如通过模拟来近似后验分布。我们将探索两种模拟技术：网格近似和马尔可夫链蒙特卡洛（MCMC）。如果做得恰当，这两种技术都会产生一个包含\(N\)个\(\theta\)值的样本，

\[\left\lbrace \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(N)} \right\rbrace ,\]

具有反映 \(\theta\) 后验模型特性的性质。在第 6 章中，我们将在熟悉的 Beta-二项 和 Gamma-泊松 模型情境下探索这些模拟技术。尽管这些模型并不 需要 模拟（我们在第一单元中可以并且已经指定了它们的后验分布），但在这些熟悉的情境中探索模拟将帮助我们建立对该过程的直觉，并让我们放心，当最终 确实 需要时，它确实有效。

  • 实现并检验使用 网格近似 模拟后验模型的局限性。
  • 探索 MCMC 后验模拟技术的基本性质，以及它们与网格近似的区别。
  • 在 Python 中实现 MCMC 模拟。
  • 学习几种 马尔可夫链诊断 方法，用于检查 MCMC 后验模拟的 质量。

6.1 网格近似

想象有一张无法完整查看的图像——你只能沿着从左到右扫过图像的网格观察到一些片段。网格越细，图像越清晰。如果网格足够细，结果就是对完整图像的极佳近似：

这就是贝叶斯网格近似背后的总体思路，目标“图像”是后验概率密度函数 \(f(\theta | y)\)。我们无需观察每一个可能的 \(\theta\) 处的 \(f(\theta | y)\) 来了解其结构。相反，我们可以在可能的 \(\theta\) 值的有限离散网格上计算 \(f(\theta | y)\)。随后，我们可以从这个离散化的概率密度函数中随机抽样，以近似完整的后验概率密度函数 \(f(\theta | y)\)。我们在此形式化这些思想并在下面应用它们。

网格近似

网格近似从后验概率密度函数 \(f(\theta|y)\) 的离散化近似中产生一个包含 \(N\) 个 独立 \(\theta\) 值 \(\left\lbrace \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(N)} \right\rbrace\) 的样本。该算法分四步进行：

  1. 定义可能的 \(\theta\) 值的离散网格。
  2. 在每个 \(\theta\) 网格值处计算先验概率密度函数 \(f(\theta)\) 和似然函数 \(L(\theta|y)\)。
  3. 通过以下步骤获得后验概率密度函数 \(f(\theta |y)\) 的离散近似：(a) 在每个 \(\theta\) 网格值处计算乘积 \(f(\theta)L(\theta|y)\)；然后 (b) 归一化 这些乘积，使它们在所有 \(\theta\) 上的总和为 1。
  4. 根据对应的归一化后验概率，随机抽取 \(N\) 个 \(\theta\) 网格值。

6.1.1 一个 Beta-Binomial 示例

为了让网格近似技术生动起来，我们来探索以下通用的 Beta-Binomial 模型（一个没有对应数据故事的模型）：

\[\begin{equation} \begin{split} Y|\pi & \sim \text{Bin}(10, \pi) \\ \pi & \sim \text{Beta}(2, 2) . \\ \end{split} \tag{6.1} \end{equation}\]

这里我们可以将 \(Y\) 解释为 10 次独立试验中的成功次数。每次试验成功的概率为 \(\pi\)，而我们关于 \(\pi\) 的先验认知由一个 \(\text{Beta}(2, 2)\) 模型刻画。假设我们观测到 \(Y = 9\) 次成功。那么根据我们在第 3 章的工作，我们知道 \(\pi\) 的更新后验模型是参数为 11（\(Y + \alpha = 9 + 2\)）和 3（\(n - Y + \beta = 10 - 9 + 2\)）的 Beta 分布：

\[\pi | (Y = 9) \sim \text{Beta}(11, 3) .\]

现在我们要请你忘记我们能够确切推导出这个后验分布。相反，我们将尝试用网格近似来近似后验。作为 第 1 步，我们需要将 \(\pi\) 在 0 到 1 上可能的连续取值划分为一个有限网格。我们从一个只包含 6 个 \(\pi\) 值的粗网格开始，\(\pi \in \{0, 0.2, 0.4, 0.6, 0.8, 1\}\)：

# 第1步：定义包含6个π值的网格
grid_data = pd.DataFrame({'pi_grid': np.linspace(0, 1, 6)})

在 第 2 步 中，我们分别使用 beta.pdf() 和 binom.pmf() 函数，针对 pi_grid 中的每个 \(\pi\)，计算 \(\text{Beta}(2,2)\) 先验概率密度函数 和给定 \(Y = 9\) 时 \(\text{Bin}(10, \pi)\) 的似然函数值：

# 第2步：在每个π值处评估先验和似然
grid_data['prior'] = beta.pdf(grid_data['pi_grid'], 2, 2)
grid_data['likelihood'] = binom.pmf(9, 10, grid_data['pi_grid'])

在 第 3 步 中，我们计算每个网格值上 likelihood 与 prior 的乘积。这给出了后验概率密度函数的一个未标准化的离散近似，其和不为 1。随后，我们将每个未标准化的后验值除以其总和，从而对这个近似进行标准化：

# 第3步：近似后验
grid_data['unnormalized'] = grid_data['likelihood'] * grid_data['prior']
grid_data['posterior'] = grid_data['unnormalized'] / grid_data['unnormalized'].sum()

# 确认后验近似之和为1
print(grid_data['unnormalized'].sum(), grid_data['posterior'].sum())

将得到的离散化后验概率密度函数四舍五入保留两位小数并绘制成下图，它让我们对实际后验概率密度函数有了一个非常粗略的了解。它表明 \(\pi\) 有大约 99% 的概率为 0.6 或 0.8，有 1% 的概率为 0.4，而在其他 3 个 \(\pi\) 网格值上的概率几乎为 0：

import pandas as pd

# 查看网格近似后验

# 绘制网格近似后验

plt.vlines(grid_data['pi_grid'], 0, grid_data['posterior'], colors='k', lw=1)
plt.xlabel('pi_grid')
plt.ylabel('后验概率')
plt.show()

你或许能预见到，当我们在步骤 4 中使用这种近似来模拟后验样本时会发生什么。每次抽样只有 6 种可能的结果，且极有可能是 0.6 或 0.8。让我们尝试一下：使用 .sample() 方法从长度为 6 的 grid_data 中有放回地抽取 10000 个值，并以离散化的后验概率作为抽样权重。

import numpy as np
import pandas as pd

# 设置随机种子
np.random.seed(84735)

# 第4步：从离散化后验中采样
post_sample = grid_data.sample(n=10000, replace=True, weights='posterior', random_state=84735)

不出所料，我们得到的 10,000 个 \(\pi\) 样本值大部分是 0.6 或 0.8，少数是 0.4，而没有低于 0.4 或高于 0.8 的值：

还记得彩虹图像吗？我们是如何通过更精细的网格来查看片段从而得到更完整的图像的呢？同样地，与其将可能的 \(\pi\) 值的 0 到 1 连续区间分割成只有 6 个值的网格，不如尝试更加合理的 101 个值的网格：\(\pi \in \{0, 0.01, 0.02, \ldots, 0.99, 1\}\)。下面展示了使用这个更精细网格的前 3 个网格近似步骤：

import pandas as pd
import numpy as np
from scipy.stats import beta, binom

# 第一步：定义 101 个 pi 值的网格
pi_values = np.linspace(0, 1, 101)
grid_data = pd.DataFrame({'pi_grid': pi_values})

# 第二步：在每个 pi 处计算先验和似然
grid_data['prior'] = beta.pdf(grid_data['pi_grid'], 2, 2)
grid_data['likelihood'] = binom.pmf(9, 10, grid_data['pi_grid'])

# 第三步：近似后验分布
grid_data['unnormalized'] = grid_data['likelihood'] * grid_data['prior']
grid_data['posterior'] = grid_data['unnormalized'] / np.sum(grid_data['unnormalized'])

6.1.2 一个 Gamma-泊松示例

作为练习，让我们将网格近似技术应用于模拟另一个贝叶斯模型的后验，这次是 Gamma-泊松模型。 设 \(Y\) 为一小时内发生的事件数量，事件发生的平均速率为每小时 \(\lambda\)。 进一步，假设我们收集了两个数据点（\(Y_1,Y_2\)），并将一个 \(\text{Gamma}(3, 1)\) 先验分布放在 \(\lambda\) 上：

\[\begin{equation} \begin{split} Y_i|\lambda & \stackrel{ind}{\sim} \text{Pois}(\lambda) \\ \lambda & \sim \text{Gamma}(3, 1) . \\ \end{split} \tag{6.2} \end{equation}\]

如果我们在第一个一小时的观测期内观测到 \(Y_1 = 2\) 个事件，在接下来的一个小时内观测到 \(Y_2 = 8\) 个事件，那么根据第 5 章中的推导，\(\lambda\) 的更新后验模型是参数为 13（\(s + \sum Y = 3 + 10\)）和 3（\(r + n = 1 + 2\)）的 Gamma 分布：

\[\lambda | ((Y_1,Y_2) = (2,3)) \sim \text{Gamma}(13, 3) .\]

为了使用网格近似模拟此后验分布，第一步是指定一个合理的离散 \(\lambda\) 值网格。 与 Beta-二项模型中受限于有限 0 到 1 区间的 \(\pi\) 参数不同，泊松参数 \(\lambda\) 理论上可以取任何非负实数值（\(\lambda \in [0,\infty)\)）。 然而，在 Gamma 先验概率密度函数和泊松似然函数图中可以看出，虽然可能，但 \(\lambda\) 值超过 15 的情况是难以置信的（图 6.5）。 考虑到这一点，我们将建立一个介于 0 到 15 之间的 \(\lambda\) 值的离散网格，实际上截断了后验的尾部。 在共同完成相应的网格近似之前，我们鼓励你先尝试下面的小测验，挑战一下自己。

# 使用 Gamma-泊松共轭先验绘制先验和后验分布
from scipy.stats import gamma
import numpy as np
import matplotlib.pyplot as plt

def plot_gamma_poisson(s=3, r=1, sum_y=10, n=2, posterior=True):
    """
    绘制 Gamma 先验和基于 Poisson 观测的后验分布。

    参数:
    s: Gamma 先验的形状参数
    r: Gamma 先验的速率参数
    sum_y: 观测值的总和
    n: 观测值的数量
    posterior: 如果为 True，则同时绘制后验分布；否则只绘制先验
    """
    lam_grid = np.linspace(0, 15, 500)
    prior_pdf = gamma.pdf(lam_grid, a=s, scale=1/r)  # scipy 使用尺度参数 scale=1/rate
    plt.plot(lam_grid, prior_pdf, label='先验 Gamma({},{})'.format(s, r))
    if posterior:
        post_shape = s + sum_y
        post_rate = r + n
        post_pdf = gamma.pdf(lam_grid, a=post_shape, scale=1/post_rate)
        plt.plot(lam_grid, post_pdf, label='后验 Gamma({},{})'.format(post_shape, post_rate))
    plt.xlabel('λ')
    plt.ylabel('密度')
    plt.legend()
    plt.title('Gamma-泊松共轭')
    plt.show()

# 调用函数
plot_gamma_poisson(s=3, r=1, sum_y=10, n=2, posterior=False)

填写下面的代码，构建对应于 (6.2) 的 Gamma-泊松后验的网格近似。在此过程中，使用 0 到 15 之间的 501 个 \(\lambda\) 值的网格。

# 网格近似
# 第 1 步：定义 501 个 λ 值的网格
import numpy as np
import pandas as pd
from scipy.stats import gamma, poisson

grid_data = pd.DataFrame({'lambda_grid': np.linspace(0, 15, 501)})

# 第 2 步：在每一个 λ 值处计算先验和似然
grid_data['prior'] = gamma.pdf(grid_data['lambda_grid'], a=3, scale=1)  # Gamma(shape=3, rate=1)
grid_data['likelihood'] = poisson.pmf(2, grid_data['lambda_grid']) * poisson.pmf(8, grid_data['lambda_grid'])

# 第 3 步：近似后验分布
grid_data['unnormalized'] = grid_data['likelihood'] * grid_data['prior']
grid_data['posterior'] = grid_data['unnormalized'] / grid_data['unnormalized'].sum()

# 设定随机种子
np.random.seed(84735)

# 第 4 步：从离散化的后验分布中抽样
post_sample = grid_data.sample(n=10000, weights='posterior', replace=True, random_state=84735)

6.1.3 局限性

有些人说，好东西终将结束。 尽管我们总体上不同意这句话，但在网格近似法上确实如此。 随着模型变得越来越复杂，网格近似法的局限性迅速显现。 例如，到第4单元结束时，我们将处理许多模型参数 \(\theta = (\theta_1, \theta_2, \ldots, \theta_k)\) 的模型。 在这样的情况下，网格近似法会受到维度灾难的困扰。 让我们回到图像近似的直觉来理解。 之前，我们假设只能沿着从左向右横扫 x 轴的网格看到图像片段。 这类似于使用网格近似法来模拟具有一个参数的模型。 现在假设我们的模型有两个参数。 或者，在图像近似的背景下，我们只能看到沿着从左向右横扫 x 轴并且从上向下扫过 y 轴的网格的片段：

当我们同时将 x 轴和 y 轴划分成网格时，图像近似中会出现更大的间隙。 为了获得更精细的近似，我们需要比仅将 x 轴划分成网格时更细的网格。 类似地，当使用网格近似法模拟多变量后验分布时，我们需要将 \(\theta = (\theta_1, \theta_2, \ldots, \theta_k)\) 的多维样本空间划分成非常非常精细的网格，以避免近似中出现大的间隙。 在实践中，这可能不可行。 在越来越细的网格上评估时，网格近似法的计算开销变得高昂。 你不能仅仅是开始模拟，起身喝杯咖啡，回来，然后砰的一声模拟就完成了。 你可能需要开始模拟，然后去参加一个长达一个月的冥想静修（以练习网格近似所需的耐心）。 马尔可夫链蒙特卡洛（MCMC）方法提供了更灵活的替代方案。

6.2 通过 PyMC 实现马尔可夫链

MCMC 方法及其名称具有一些历史意义。 MCMC 中的马尔可夫链部分以俄罗斯数学家安德雷·马尔可夫（Andrey Markov，1856–1922）命名。 蒙特卡洛部分的词源则更为可疑。 作为20世纪40年代绝密核武器项目的一部分，斯坦尼斯瓦夫·乌拉姆（Stanislav Ulam）、约翰·冯·诺依曼（John von Neumann）及其在洛斯阿拉莫斯国家实验室的合作者使用马尔可夫链来模拟和更好地理解中子输运 (Eckhardt 1987)。 洛斯阿拉莫斯团队将其工作代号称为“蒙特卡洛”，据说这个选择受到法国里维埃拉奢华的蒙特卡洛赌场的启发。 是什么激发了随机模拟方法与赌场之间的联系？ 我们在这里无法帮你解答。

图 6.7：田纳西州橡树岭一处铀处理工厂的广告牌，橡树岭是洛斯阿拉莫斯的姐妹站点。图片：https://commons.wikimedia.org/wiki/File:Oak_Ridge_Wise_Monkeys.jpg

如今，“马尔可夫链蒙特卡洛”是指使用蒙特卡洛项目首创的方法，应用马尔可夫链来模拟概率模型。 与网格近似法相比，MCMC 模拟方法可以扩展到更复杂的贝叶斯模型。 但这种灵活性并非免费。 与网格近似样本一样，MCMC 样本并非直接从后验概率密度函数 \(f(\theta|y)\) 中抽取。 但与网格近似样本不同的是，MCMC 样本甚至不是独立的——每个后续样本值直接依赖于前一个值。 这一特征由“链”这一术语体现。 具体来说，令 \(\left\lbrace \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(N)} \right\rbrace\) 是长度为 \(N\) 的 MCMC 样本，或称马尔可夫链。 在构建这条链时，\(\theta^{(2)}\) 是从某个依赖于 \(\theta^{(1)}\) 的模型中抽取的，\(\theta^{(3)}\) 是从某个依赖于 \(\theta^{(2)}\) 的模型中抽取的，\(\theta^{(4)}\) 是从某个依赖于 \(\theta^{(3)}\) 的模型中抽取的，以此类推，链不断增长。 一般来说，第 \((i + 1)\) 个链值 \(\theta^{(i+1)}\) 是从一个依赖于数据 \(y\) 和前一个链值 \(\theta^{(i)}\) 的模型中抽取的，其条件概率密度函数为

\[f\left(\theta^{(i + 1)} \; | \; \theta^{(i)}, y\right) .\]

关于链值之间的这种依赖性，有几点需要注意。 首先，根据马尔可夫性质，\(\theta^{(i+1)}\) 仅通过最近的值 \(\theta^{(i)}\) 依赖于前面的链值：

\[f\left(\theta^{(i + 1)} \; | \; \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(i)}, y\right) = f\left(\theta^{(i + 1)} \; | \; \theta^{(i)}, y\right) .\]

例如，因为 \(\theta^{(i+i)}\) 依赖于 \(\theta^{(i)}\)，而 \(\theta^{(i)}\) 又依赖于 \(\theta^{(i-1)}\)，所以 \(\theta^{(i+i)}\) 也依赖于 \(\theta^{(i-1)}\)。然而，如果我们知道了 \(\theta^{(i)}\)，那么 \(\theta^{(i-1)}\) 对 \(\theta^{(i+1)}\) 就没有影响了 —— 模拟 \(\theta^{(i+1)}\) 所需的唯一信息就是 \(\theta^{(i)}\) 的值。此外，链中的每个值都可能来自不同的模型，而且这些模型中 没有一个 是目标后验分布。也就是说，生成马尔可夫链值所用的概率密度函数并不等同于后验概率密度函数：

\[f\left(\theta^{(i + 1)} \; | \; \theta^{(i)}, y\right) \ne f\left(\theta^{(i + 1)} \; | \; y\right) .\]

马尔可夫链蒙特卡洛（MCMC）

MCMC 模拟会产生一条由 \(N\) 个 相依 的 \(\theta\) 值组成的 链，\(\left\lbrace \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(N)} \right\rbrace\)，这些值 并非 从后验概率密度函数 \(f(\theta|y)\) 中抽取。

使用一个并不来自后验的相依样本来近似后验，这似乎有些奇怪。但从数学魔法看，借助合理的 MCMC 算法，这是可行的。难点在于这些算法的学习曲线比网格近似技术更陡。我们将在第 7 章简要介绍这些细节。尽管彻底阅读第 7 章并牢固掌握这些细节是理想的，但现有越来越多的 MCMC 计算资源可以为我们完成繁重的工作。在本书中，我们将使用 PyMC 软件包进行 MCMC 模拟 (Guo et al. 2020)，它结合了 Python 的强大功能与 Stan 引擎。你将在第 6 章深入了解 PyMC 的细节，逐行构建代码，从而熟悉 PyMC 模拟的关键要素。从第 9 章开始，你将利用互补的 Bambi 软件包，它提供了模拟广泛的贝叶斯 应用回归模型 (arm) 的快捷方式。

6.2.1 一个 Beta-二项示例

所有 PyMC 分析都包含两个基本步骤：（1）用 PyMC 语法 定义贝叶斯模型结构；（2）模拟后验分布。让我们在等式 (6.1) 定义的 Beta-二项模型的背景下探讨这些步骤，从 步骤 1 开始。在定义该模型的结构时，我们必须指定其依赖的三个方面：

  • data
    数据 \(Y\) 是 10 次试验中观测到的成功次数。由于 PyMC 不是读心者，我们必须指定 \(Y\) 是一个介于 0 到 10 之间的 整数。

  • parameters
    模型依赖于参数 \(\pi\)，在 PyMC 中记为 pi。我们必须指定 \(\pi\) 可以是 0 到 1 之间的任何实数。

  • model
    模型由数据 \(Y\) 的 \(\text{Bin}(10, \pi)\) 似然和 \(\pi\) 的 Beta\((2,2)\) 先验定义。我们使用 pm.Binomial 和 pm.Beta 来指定它们。

下面我们将这个 Beta-二项结构转换为 PyMC 语法，并存储为模型对象 bb_model。 鼓励你暂停并检查代码，注意它与上述三个方面如何对应。

import pymc as pm

# 定义模型：Y ~ binomial(10, pi), pi ~ Beta(2,2)
with pm.Model() as bb_model:
    pi = pm.Beta('pi', alpha=2, beta=2)
    Y = pm.Binomial('Y', n=10, p=pi, observed=9)

在步骤 2，我们使用 pm.sample() 函数模拟后验。粗略地说，pm.sample() 设计并运行一个 MCMC 算法，从 Beta-二项后验中产生近似样本。

由于 pm.sample() 既要为给定模型识别合适的 MCMC 算法，又要将该算法应用于我们的数据，因此对每个新模型的模拟都会相当慢。

import pymc as pm

# 模拟后验
with bb_model:
    idata = pm.sample(draws=5000, tune=5000, chains=4, random_seed=84735)

注意，pm.sample() 需要两种参数。首先，我们必须通过模型对象指定模型信息：

  • 模型 = 定义模型的 PyMC 模型对象（这里为 bb_model）
  • 数据 = 观测数据的列表（这里 Y = 9），已在模型中通过 observed 参数传递。

其次，我们必须通过三个额外的参数来指定所需的马尔可夫链信息：

  • chains 参数指定运行多少条并行马尔可夫链。我们在这里运行四条链，因此得到四组不同的 \(\pi\) 值样本。我们将在第 6.3 节讨论这一选择。

  • iter 参数指定每条马尔可夫链所需的迭代次数，即链的长度。默认情况下，前一半迭代会被丢弃，作为“预热”（burn-in）或“预热”样本（详见下文），后一半则保留为最终的马尔可夫链样本。

  • 要为 PyMC 模拟设置随机数生成种子，我们在 pm.sample() 函数内部使用 random_seed 参数。

结果存储在 bb_sim 中，是一个 InferenceData 对象。该对象包含四条并行马尔可夫链，每条运行 10,000 次迭代。在丢弃所有四条链的前 5,000 次迭代后，我们得到四个大小为 5,000 的独立马尔可夫链样本，合并后的马尔可夫链样本量为 20,000。

预热（Burn-in）

如果你曾经摊过煎饼或可丽饼，你就会知道第一张总是最差的——锅还没达到理想温度，你还没掌握好面糊的用量，也需要更多时间来练习翻面的技巧。MCMC 链与之类似。由于对要模拟的后验没有直接了解，马尔可夫链可能一开始会从我们感兴趣的参数（比如 \(\pi\)）的不合理值开始采样。但最终它会学习并开始生成能够模仿后验随机样本的值。就像我们可能需要扔掉第一张煎饼一样，我们可能也想丢弃在学习期内产生的马尔可夫链值——把它们留在样本中可能导致后验近似质量不佳。因此，“预热”是指丢弃马尔可夫链前一部分值的做法。

提取并展示每条并行链的前四个 \(\pi\) 值：

# 提取后验中 pi 参数的前4个迭代（每条链）
pi_draws = idata.posterior['pi'].values  # 形状 (chains, draws)
print(pi_draws[:, :4].T)  # 转置，行=迭代，列=链

6.2.2 一个 Gamma-泊松示例

为了进一步练习 PyMC 和 MCMC 模拟，我们使用这些工具来近似对应于 (6.2) 的 Gamma-泊松后验，给定观测数据 \((Y_1,Y_2) = (2,8)\)。 回想一下，这涉及两个步骤。 在 步骤 1 中，我们定义 Gamma-泊松模型结构，指定其所依赖的三个方面：data、parameters 和 model。 在 步骤 2 中，我们模拟后验。 我们鼓励你先自己尝试挑战一下。 你的代码将用到两个你尚未见过的术语，但你可能猜到如何使用：Poisson() 和 Gamma()。 此外，你的代码将包含一个关于 \((Y_1,Y_2)\) 变量和观测的向量，而不是单个变量 \(Y\)。

请补全下面的代码，以构建与公式 (6.2) 对应的 Gamma-泊松后验分布的 MCMC 近似。在此过程中，运行 4 条平行链，每条链迭代 10,000 次（每条链的样本量为 5,000）。

import pymc as pm
import arviz as az

# 步骤1：定义模型
with pm.Model() as gp_model:
    # 先验：lambda ~ Gamma(3, 1)
    lam = pm.Gamma('lam', alpha=3, beta=1)
    # 似然：Y ~ Poisson(lambda)，观测数据为 [2, 8]
    Y = pm.Poisson('Y', mu=lam, observed=[2, 8])

# 步骤2：模拟后验
with gp_model:
    # 四条链，每条链采样 5000 次（预热 5000 次），设置随机种子

                         random_seed=84735, return_inferencedata=True)

  • data

6.3 马尔可夫链诊断

模拟真是太棒了。 即使我们能推导出贝叶斯后验，我们也可以使用后验模拟来验证我们的工作并提供直观理解。 更重要的是，当我们无法推导出后验时，模拟样本能提供关键的近似。 在上面的MCMC示例中，我们看到马尔可夫链遍历我们参数（\(\pi\) 或 \(\lambda\)）的样本空间，并且最终模仿一个随机样本，该样本最终收敛到后验。 “近似”和“收敛”是这里的关键词——模拟并不完美。 这引出了以下问题：

  • 一个好的马尔可夫链看起来是什么样的？
  • 我们如何判断马尔可夫链样本是否产生了对后验的合理近似？
  • 马尔可夫链的样本量应该多大？

回答这些问题既是一门艺术，也是一门科学。 没有一刀切的魔法公式能给出确定答案。 相反，通过经验，你会对“好的”马尔可夫链长什么样以及如何修复“坏的”马尔可夫链有所感觉。 在本节中，我们将关注几个能让我们入门的可视化诊断工具：踪迹图和并行链。 这些可以用一些数值诊断来跟进和补充：有效样本量、自相关性和R-hat（\(\hat{R}\)）。 使用这些诊断应该整体性地进行。 由于没有单一的可视化或数值诊断是一刀切的，当一起考虑时，它们能提供对马尔可夫链质量的更全面图景。 此外，还有其他优秀的诊断方法存在。 我们在此专注于那些常见且易于在我们将要使用的软件包中实现的诊断。

6.3.1 检查踪迹图

重新检查图 6.8 中 Beta-二项模拟的踪迹图。 这些是我们希望踪迹图呈现的教科书示例。 主要地，它们看起来像是一堆白噪声，没有可辨别的趋势或值得注意的现象。 这种“空白”意味着链是稳定的。 相反，图 6.12 中展示的同一 Beta-二项模型的假设马尔可夫链，则说明了可能让我们警觉的潜在链行为。

图 6.12：两条假设马尔可夫链的轨迹图（左）和对应的密度图（右）。这些展示了“坏”的马尔可夫链可能的样子。叠加的黑色曲线（右）表示目标后验概率密度函数 Beta(11,3)。

首先，考虑图 6.12（左）中的轨迹图。 链 A 的下降趋势表明其在 5000 次迭代后尚未稳定——它尚未“找到”或还不知道如何探索 \(\pi\) 的后验合理值范围。 下降趋势也暗示链值之间存在强相关性——它们看起来不像独立噪声。 所有这一切都说明链 A 是混合缓慢的。 这很糟糕。 尽管马尔可夫链本身具有依赖性，但它们的行为越类似于快速混合（噪声）的独立样本，得到的后验近似误差就越小（粗略地说）。 链 B 表现出不同的问题。 正如轨迹图中两条完全平坦的线所示，它在访问较小的 \(\pi\) 值时往往会卡住。

图 6.12（右）中的密度图证实，这两条看起来古怪的链都导致了一个严重问题：它们产生了 Beta(11,3) 后验分布（黑色叠加）的不良近似，从而得出误导性的后验结论。 考虑链 A。 由于混合非常缓慢，它在最初的 5000 次迭代中仅探索了 \(\pi\) 值大约在 0.6 到 0.9 的粗略范围。 因此，其后验近似高估了这个范围内 \(\pi\) 值的合理性，而完全低估了范围外值的合理性。 接下来，考虑链 B。 在卡住时，链 B 对后验 \(\pi\) 值左尾的一些值进行了过度采样。 这种现象导致了后验近似中的错误峰值。

在实践中，当我们无法指定后验分布并希望对其进行近似时，我们会运行 PyMC 模拟。 这意味着我们没有机会将模拟结果与“真实”后验进行比较。 这就是诊断如此重要的原因。 如果我们看到如图 6.12 中那样的糟糕轨迹图，我们可以立即采取一些步骤：

  1. 检查模型。所假设的先验分布和数据模型是否合适？
  2. 运行更多的链迭代。一些不良的短期链趋势可能会在长期内消除。

我们将在本书中练习更微妙的第 1 步。第 2 步很简单，尽管需要额外的计算时间。

6.3.3 计算有效样本量 & 自相关

回想一下第 6.3.1 节，一个相依的马尔可夫链越表现出类似独立样本的行为，其后验近似的误差就越小（粗略地说）。尽管踪迹图能提供关于这种行为的一些直观信息，但补充的数值评估可以提供更细致的信息。首先，回想一下我们的马尔可夫链模拟 bb_sim 产生了从后验分布以外的模型中采样的 20,000 个 \(\pi\) 的相依值。然后我们使用这条链来近似 \(\pi\) 的后验分布。知道如果使用直接从后验分布抽取的 20,000 个独立样本值，近似的误差可能会更小，这就引出了以下问题：相对而言，需要多少个独立样本值才能产生同等准确的后验近似？有效样本量比率提供了答案。

有效样本量比率

设 \(N\) 表示一条相依马尔可夫链的实际样本量或长度。该链的有效样本量 \(N_{eff}\) 量化了需要多少个独立样本才能产生同等准确的后验近似。\(N_{eff}\) 越大越好，但通常马尔可夫链近似的准确度仅相当于一个较小的独立样本。也就是说，通常有 \(N_{eff} < N\)，因此有效样本量比率小于 1：

\[\frac{N_{eff}}{N}\]

对于解释这个比率没有魔法定律，它应与其他诊断工具（如踪迹图）一起使用。尽管如此，如果有效样本量比率小于 0.1，即有效样本量 \(N_{eff}\) 小于实际样本量 \(N\) 的 10%，我们可能会对这条马尔可夫链产生怀疑。

利用 ArviZ 库可以方便地计算我们马尔可夫链样本中 pi 值的有效样本量比率。这里，使用长度为 20000 的马尔可夫链来近似 \(\pi\) 的后验分布，其精度大致相当于只使用了 34% 的独立值。换句话说，我们这 20000 个马尔可夫链值大约只相当于 6800 个独立样本 (0.34 \(\cdot\) 20000)。由于这个比率高于 0.1，我们不必担心。

# 计算有效样本量比率
n_chains = bb_sim.posterior.dims["chain"]
n_draws = bb_sim.posterior.dims["draw"]
ess = az.ess(bb_sim, var_names=["pi"]).values
neff_ratio = ess / (n_chains * n_draws)
neff_ratio

自相关提供了另一个衡量马尔可夫链是否足够模拟独立样本行为的指标。强自相关或依赖性是个坏兆头——它与较小的有效样本量比率相伴而生，从而警示我们得到的后验近似可能不可靠。我们在图 6.12 的链 A 中看到了一些证据。现在，由于马尔可夫链的简单构造，链值之间必然存在一定的自相关——一个链值 (\(\pi^{(i)}\)) 依赖于前一个链值 (\(\pi^{(i-1)}\))，后者又依赖于更前一个链值 (\(\pi^{(i-2)}\))，依此类推。这种依赖链条也意味着每个链值在某种程度上依赖于所有先前的链值。例如，由于 \(\pi^{(i)}\) 依赖于 \(\pi^{(i-1)}\)，而 \(\pi^{(i-1)}\) 又依赖于 \(\pi^{(i-2)}\)，因此 \(\pi^{(i)}\) 也依赖于 \(\pi^{(i-2)}\)。然而，这种依赖性或自相关会逐渐减弱。这类似于托布勒的地理学第一定律：万物皆有联系，但近处的事物比远处的事物联系更紧密。因此，通常情况是，链值 \(\pi^{(i)}\) 与前一个值 (\(\pi^{(i - 1)}\)) 的相关性强于与 100 步之前的链值 (\(\pi^{(i-100)}\)) 的相关性。

自相关

滞后 1 自相关衡量的是相差一个“步”的马尔可夫链值对之间的相关性（例如，\(\pi^{(i)}\) 和 \(\pi^{(i-1)}\)）。 滞后 2 自相关衡量的是相差两个“步”的马尔可夫链值对之间的相关性（例如，\(\pi^{(i)}\) 和 \(\pi^{(i-2)}\)）。 以此类推。

让我们把这些概念应用到 bb_sim 分析中。查看图 6.15 中我们模拟结果的轨迹图和自相关图。（为简单起见，我们只展示四条并行链中一条链的结果。）

快速混合与缓慢混合的马尔可夫链

快速混合链表现出类似于独立样本的行为：链在后验可信值的范围内“快速”移动，链值之间的自相关迅速下降，有效样本量比值相当大。 缓慢混合链则不具备独立样本的特征：链在后验可信值的范围内“缓慢”移动，链值之间的自相关下降非常缓慢，有效样本量比值很小。

那么，如果我们的链是令人畏惧的缓慢混合链，我们该怎么办？ 第一个，也是我们认为最好的可能解决方案是久经考验的“运行更长的链”。 即使是缓慢混合的链，如果我们运行足够多的迭代次数，也最终能产生良好的后验近似。 另一种常见的方法是稀释马尔可夫链。 例如，从原始的5000个\(\pi\)值的链中，我们可能保留每隔一个值并丢弃其余的：\(\left\lbrace \pi^{(2)}, \pi^{(4)}, \pi^{(6)}, \ldots, \pi^{(5000)} \right\rbrace\)。 或者，我们可能保留每隔十个链值：\(\left\lbrace \pi^{(10)}, \pi^{(20)}, \pi^{(30)}, \ldots, \pi^{(5000)} \right\rbrace\)。 通过丢弃中间的抽取，我们移除了低滞后处的强相关性。 例如，\(\pi^{(20)}\)与稀释链中的前一个值（\(\pi^{(10)}\)）的相关性要小于与原始链中的前一个值（\(\pi^{(19)}\)）的相关性。

仅作说明，我们使用采样函数中的 thin 参数将原始的 bb_sim 链稀释为每隔十个值。 结果，自相关下降得更早一些，在1个滞后之后而不是5个（图6.17）。 但是，由于这已经是一个快速混合模拟，自相关迅速下降且有效样本量比值相对较高，因此自相关的这点微小改善并不值得我们丢失的信息。 我们本有5000个链值来近似后验，现在只有500个值。

import pymc as pm
import arviz as az

# 构建 Beta-二项模型
with pm.Model() as bb_model:
    pi = pm.Beta('pi', alpha=2, beta=2)    # 先验分布 Beta(2,2)
    Y = pm.Binomial('Y', n=10, p=pi, observed=9)   # 观测到 9 次成功（总试验次数 10）

# MCMC 采样：4 条链，每条链 10000 次采样
with bb_model:
    thinned_trace = pm.sample(draws=10000, chains=4, random_seed=84735, progressbar=False)

# 薄化：每 10 个样本保留 1 个
thinned_trace = thinned_trace.thin(10)

# 检查结果
az.plot_trace(thinned_trace, var_names=["pi"])
az.plot_autocorr(thinned_trace, var_names=["pi"])

我们类似地将缓慢混合链稀释为每隔十个值（图6.18）。 得到的链在迹图中仍显示出缓慢混合的趋势，但自相关比稀释前下降得更快。 这很好，但值得损失90%的原始样本值吗？ 我们不太确定。

图6.18：\(\pi\)的缓慢混合马尔可夫链的迹图（左）和自相关图（右），已稀释为每隔十个值。

在决定是否稀释马尔可夫链时，需要谨慎权衡。 降低自相关的好处并不一定超过宝贵链值的损失。 也就是说，具有较强自相关的5000个马尔可夫链值可能比具有较弱自相关的500个链值产生更好的后验近似。 稀释的有效性还部分取决于用于构建马尔可夫链的算法。 例如，本书中使用的PyMC和ArviZ等库采用了高效的哈密顿蒙特卡洛算法。 因此，在当前的采样函数帮助文件中，库作者建议不要稀释，除非你的模拟占用了太多内存。

6.3.4 计算R-hat

正如有效样本量和自相关为评估马尔可夫链在多大程度上表现得像独立样本提供了可视迹线图诊断的数值补充，分裂-\(\hat{R}\) 指标（或简称 R-hat）为并行链的可视比较提供了数值补充。 回顾第 6.3.2 节，我们不仅希望模拟中的每条单独的马尔可夫链都稳定，还希望各条并行链之间保持一致性。 R-hat 通过比较所有链合并后的采样 \(\pi\) 值变异性与每条链内部的变异性来处理这种一致性。 在给出更正式的定义之前，请先凭直觉思考。40

图 6.19 显示了 bb_sim（顶行）以及一个较差的假设替代方案（底行）的模拟结果。 根据这些图中的模式，你认为什么是“好”的马尔可夫链模拟的标志？

  1. 任何单条链内部的 \(\pi\) 值变异性小于所有链合并后的 \(\pi\) 值变异性。
  2. 任何单条链内部的 \(\pi\) 值变异性与所有链合并后的 \(\pi\) 值变异性相当。

图 6.19：bb_sim（顶行）和一个假设的替代模拟（底行）的结果。包括四条并行链的迹线图（左）、每条链的密度图（中）以及合并链的密度图（右）。

要回答这个小测验，我们来仔细分析图 6.19。 根据第 6.3.2 节中学到的知识，我们可以看出 bb_sim 优于替代方案——其并行链表现出相似的特征，并产生相似的后验近似。 特别是，每条链内部的 \(\pi\) 值变异性几乎完全相同（顶行中图）。 因此，所有链合并后的 \(\pi\) 值变异性（顶行右图）与各条链的变异性相似。 相比之下，注意到替代模拟中的四条并行链产生了相互矛盾的后验近似（底行中图），因此当我们将这些链合并时，得到的后验近似不稳定且质量较差（底行右图）。 因此，所有链合并后的 \(\pi\) 值范围和变异性，远大于任何单条链内部的 \(\pi\) 值范围和变异性。

综上所述，我们直观地理解了所有链合并后的变异性与各条并行链内部变异性之间关系的重要性。 具体而言：

  • 在一个“好”的马尔可夫链模拟中，所有并行链合并后的变异性与任何单条链内部的变异性大致相当；

  • 在“糟糕的”马尔可夫链模拟中，所有平行链合并后的变异可能会超过每条链内部的典型变异。

我们可以使用 R-hat 来量化合并链变异与链内变异之间的关系。这里给出 R-hat 的直观定义。更详细的定义参见 Vehtari 等（2021）。想了解更多关于有效样本量与 R-hat 之间有趣联系的内容，请参阅 Vats 和 Knudson（2018）！

R-hat

考虑参数 \(\theta\) 的马尔可夫链模拟，该模拟使用了四条平行链。令 \(\text{Var}_\text{combined}\) 表示所有四条链合并的 \(\theta\) 变异，\(\text{Var}_\text{within}\) 表示任一单独链内的典型变异。R-hat 指标计算这两部分变异的比值：

\[\text{R-hat} \approx \sqrt{\frac{\text{Var}_\text{combined}}{\text{Var}_\text{within}}} .\]

理想情况下，\(\text{R-hat} \approx 1\)，反映平行链之间的稳定性。相反，\(\text{R-hat} > 1\) 则表明不稳定，合并链的变异超过链内变异。虽然没有绝对的标准，但 R-hat 比值大于 1.05 时应当引起对模拟稳定性的警惕。

我们可以使用 ArviZ 库中的 rhat() 函数来计算模拟的 R-hat 比值：

az.rhat(bb_trace, var_names=["pi"])

反映我们观察到四条平行链之间和内部的变异相当，bb_sim 的 R-hat 值实际上等于 1。与之相反，图 6.19 中展示的糟糕的模拟假设中，R-hat 值是 5.35。也就是说，所有合并链的方差是各链内部典型方差的 5 倍以上。这远远超过了 1.05 的警戒线，充分表明那假设的平行链没有产生一致的后验近似，因此模拟不稳定。

6.4 本章概要

随着我们的贝叶斯模型变得更加复杂，后验分布将变得难以甚至不可能明确表达。在第 6 章中，你学习了两种模拟技术用于近似后验分布：网格近似和马尔可夫链蒙特卡洛。两种技术都生成了 \(N\) 个 \(\theta\) 值的样本，

\[\left\lbrace \theta^{(1)}, \theta^{(2)}, \ldots, \theta^{(N)} \right\rbrace .\]

然而，这些样本的性质不同：

  • 网格近似从后验概率密度函数 \(f(\theta|y)\) 的离散化近似中抽取 \(\theta^{(i)}\) 的独立样本。这种方法非常直接，但在更复杂的模型设置下会失效。
  • 马尔可夫链蒙特卡洛提供了更灵活的替代方案。尽管MCMC生成的 \(\theta^{(i)}\) 样本是相依的，且并非直接从后验概率密度函数 \(f(\theta|y)\) 中抽取，但只要链长 \(N\) 足够大，该样本就能很好地模拟后验模型。

最后，你学习了一些用于检查模拟结果质量的MCMC诊断方法。简而言之，我们可以通过视觉检查多条平行链的轨迹图和密度图，以评估模拟的稳定性和混合性：

# 诊断图
az.plot_trace(my_trace, var_names=["___"])
az.plot_density(my_trace, var_names=["___"])

我们还可以用数值诊断指标来补充这些视觉诊断，例如有效样本量比例、自相关和R-hat：

import arviz as az

# 有效样本量比率
az.ess(my_sim, var_names=["___"])

# 自相关图
az.plot_autocorr(my_sim, var_names=["___"])

# R-hat 诊断
az.rhat(my_sim, var_names=["___"])
