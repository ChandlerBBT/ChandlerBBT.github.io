第2章 单个证券收益率

在本章中，我们关注证券价格的百分比变化，这被称为证券的收益率。从投资的角度来看，证券的收益率有时比我们投资金额的美元变化更为重要。为了说明原因，假设有人告诉你他们通过投资某只股票在1年内赚了500美元。仅凭这一信息并不能告诉我们500美元是否是一个良好的投资回报，因为我们还需要知道这个人投入了多少资金来赚取这500美元。如果初始投资是1000美元，那么赚500美元是非常出色的，因为这相当于1年内50%的收益率。然而，如果初始投资是100,000美元，那么赚500美元仅相当于0.5%的收益率。作为对比，2019年底的1年期美国国债收益率大约为1.5%。如果无风险投资的收益率为1.5%，那么风险投资仅获得0.5%的收益率就不那么令人印象深刻了。

当我们把收益率描述为价格的百分比变化时，实际上并未描述全貌。许多证券会提供中间现金流，例如股票的股息和债券的息票。这些中间现金流可以再投资回该证券或其他投资。在实践中，对于某些证券，一些经纪商（例如Schwab）允许投资者勾选一个选项，以便将股息再投资。因此，中间现金流的再投资并不仅仅是一个理论概念。假设中间现金流再投资的收益率称为总收益率。由于总收益率衡量的是我们在持有证券的整个期间内获得的回报，因此这种收益率也称为持有期收益率。当需要加以区分时，我们将不包含中间现金流再投资的收益率称为价格收益率。

我们首先在2.1节和2.2节中说明如何计算价格收益率和总收益率。然后，出于统计和编程方面的考虑，我们在2.3节中说明如何计算对数总收益率。接着在2.4节中，我们介绍处理数据中异常值的两种可能方法：缩尾处理和截尾处理。在2.5节中，我们考察较长持有期内的收益率，并展示如何使用算术收益率和对数收益率将日收益率累积为多日收益率。

2.1 价格收益率

仅基于证券收盘价的美元变动称为资本利得，而证券收盘价的百分比变动则是其价格收益率。价格收益率是在某个投资期限（例如1天、1个月、1年等）内衡量的。期限的长度取决于应用场景，但收益率计算应保持一致，使得一年内所有日收益率的累计值等于年收益率。

日价格收益率是今日证券价格相对于昨日价格的百分比变化，即：

\[ R^{price}_t = \frac{P_t - P_{t-1}}{P_{t-1}} = \frac{P_t}{P_{t-1}} - 1, \tag{2.1} \]

其中，\(R^{price}_t\) 是第 t 天的价格收益率，\(P_t\) 是第 t 天的证券价格，\(P_{t-1}\) 是前一个交易日证券的价格。

在本节中，我们将以 AAPL 为例，因为 AAPL 派发股息，这样我们可以比较价格收益率和总收益率的结果。下面展示如何计算 AAPL 从 2015 年至 2019 年的日价格收益率。

关于Yahoo Finance股票拆分与收盘价的说明

使用现实世界数据的问题在于它会变化。在本书的第1版中，Yahoo Finance的收盘价未进行拆股调整。然而，在撰写第2版时，Yahoo Finance的收盘价现已针对拆股进行了调整。因此，之前使用Yahoo Finance收盘价计算收益率的问题已不复存在。但是，使用拆股调整后的收盘价可能会带来其他问题。例如，如果我们想计算公司的市值，使用的流通股数应与使用的价格一致。否则，我们会计算出完全错误的市值。

以Netflix (NFLX) 2015年7月15日的7:1股票拆分为例。Yahoo Finance目前报告拆股前一日（2015年7月14日）的（拆股调整后）收盘价为$100.37。拆股前价格超过$700。更令人困惑的是，NFLX截至2015年6月30日（即拆股后）的10-Q报告显示，截至2015年6月30日，NFLX有4.26亿流通股。然而，进一步深究，我们知道NFLX拆股前约有6000万流通股。因此，使用拆股前数据计算NFLX市值为每股拆股前价格$700乘以6000万拆股前流通股。另一方面，使用拆股后数据，NFLX市值为拆股后价格$100乘以约4.2亿拆股后流通股。混合使用拆股前后数字会导致截然不同且错误的结果。

2.2 总收益率

投资者持有股票所获得的收益并不局限于证券价格的变化。对于支付股息的公司，在除息日之前持有股票的股东会收到现金支付，并能够将其再投资。因此，股东从支付股息的股票中获得的总收益率包括她所持股票价格的变化，以及股息产生的任何收入以及在除息日这些股息的再投资。包含股息再投资的收益率称为持有期收益率或总收益率。

持有证券的总收益率计算如下：

\[ R_t = \frac{P_t + CF_t - P_{t-1}}{P_{t-1}} = \left[ \frac{P_t}{P_{t-1}} - 1 \right]_{\text{资本增值}} + \frac{CF_t}{P_{t-1}}_{\text{现金流收益率}}, \qquad (2.2) \]

其中 \(CF_t\) 是日期 \(t\) 的现金流（例如股息），所有其他项的定义与式 (2.1) 相同。第一项代表资本增值，第二项代表股息收益率。式 (2.2) 中的分解也有助于识别收益率的来源，因此偏好资本利得或股息（例如出于税收目的）的投资者可以更好地评估该投资对其特定投资目标的吸引力。由于我们通常会使用总收益率，因此我们不添加上标来标记总收益率的计算。

对于日总收益率计算，股息收益率在非除息日为零。这有两个含义。首先，在大多数日子中，价格收益率和总收益率相同，因为在这些日期我们只有资本增值的变化。其次，对于不支付股息的股票，价格收益率和总收益率相同。

2.3 对数总收益率

在前一节中使用公式(2.2)或Delt()计算的收益率被称为算术收益率（arithmetic returns）或简单收益率（simple returns）。在本节中，我们将展示如何计算对数收益率（logarithmic returns）或对数收益率（log returns）。对数收益率广泛应用于衍生品定价及其他金融领域。此外，当我们稍后在本章中计算多期收益率时，我们将展示使用对数收益率计算累计收益率相对更简单。

对数收益率 \( r_t \) 的计算公式为： \[ r_t = \ln\left(\frac{P_t}{P_{t-1}}\right) = \ln(1+R_t) = \ln P_t - \ln P_{t-1}, \qquad (2.3) \] 其中 \(\ln\) 为自然对数算子，其余变量的定义与式(2.2)相同。因此，我们可以通过取对数价格的差值来计算对数收益率。此外，由于我们是使用调整收盘价计算对数收益率，因此不会在对数收益率变量 \(r\) 上添加下标。

我们现在计算AAPL股票从2015年1月至2019年12月的对数总收益率。为应用公式(2.3)，我们对调整收盘价应用 log() 函数，然后使用 diff() 计算差值。

何时应使用缩尾处理（winsorization）而不是截尾处理（truncation），或反之，并没有固定的规则。选择取决于我们要进行的分析。有时人们更倾向于缩尾处理，因为这样仍能保留观测数量，但这样做等于承认我们不相信这些值应该如此高或如此低。然而，如果缩尾处理或截尾处理的选择导致结果发生实质性变化，那么我们应该确保理解差异的来源，并且确信我们所做的选择不会影响结果的可靠性。

在评估投资时，我们通常关心投资在特定时间范围内的表现。换句话说，我们感兴趣的是累积多日收益率（cumulative multi-day returns）。我们可能想知道投资在过去一周、过去一个月或过去一年的收益率。为了充分捕捉股息再投资的效果，我们应该计算每日收益率，并将这些收益率串联成更长的周期。否则，如果我们仅使用投资期初和期末的价格并加上股息来应用式（2.2），那么我们就假设股息是在期末收到的，并且这些股息没有获得额外收益。如果假设股息未再投资而是放在床垫下，那么这种计算可能是合适的。然而，我们认为更合理的做法是将股息再投资于与初始投资具有类似风险-收益特征（risk-return profile）的证券，而满足这一条件的证券正是产生这些股息的同一只证券。换句话说，当我们收到股息时，我们会将该金额再投资于该股票。该股票未来的收益率决定了再投资的股息是获得了正收益还是负收益。

现在让我们通过一个例子来了解如何累积多日收益率。假设我们想知道，如果在2014年12月31日以收盘价买入AMZN股票，到2019年底，这笔投资会带来多少收益。我们将展示如何使用算术收益率（arithmetic returns）和对数收益率（logarithmic returns）实现这一计算，并说明当一致地实施时，两种类型的收益率会得出相同的结果。然而，有些人可能会发现对数收益率的编程稍微简单一些。

2.5.1 累积算术收益率

为了将多日的算术收益率连接起来，我们需要计算日总收益率的乘积。若 \(R_t\) 为第 \(t\) 天的净收益率，则第 \(t\) 天的总收益率等于 \(1 + R_t\)。换句话说，对于 2 天累积收益率，我们有 \(R_{1,2} = (1 + R_1) \times (1 + R_2)\)。对于 3 天累积收益率，我们取 \(R_{1,3} = (1 + R_1) \times (1 + R_2) \times (1 + R_3)\)。因此，我们可以将这一计算推广到 \(T\) 天的投资期限：

\[ R_{1,T} = (1 + R_1) \times (1 + R_2) \times \cdots \times (1 + R_T) = \prod_{t=1}^{T} (1 + R_t). \qquad (2.4) \]

步骤 1：计算总收益率 由于我们已经在上文中计算了每日算术总收益率，因此我们可以直接使用 tot.ret 数据集进行此步骤。我们首先通过将 tot.ret 中的值加 1 来计算总收益率。

2.5.2 累积对数收益率

计算多期收益率的另一种方法是取每日对数收益率之和。即

\[ r_{1,T} = \ln\left((1+R_1)(1+R_2)\cdots(1+R_T)\right) = r_1 + r_2 + \cdots + r_T = \sum_{t=1}^T r_t. \tag{2.5} \]

由于我们已经在 log.rets 中计算了 AMZN 的对数收益率，计算累积对数收益率仅需一步。我们只需对这段时间内的每日对数收益率求和。使用 sum() 得到 AMZN 在 2015 年 1 月至 2019 年 12 月期间的累积对数收益率为 106.3%。
